## 🍎 Apple Silicon & Local AI

### Orion: First Open End-to-End ANE Training + Inference System (arXiv)
A new Apple ML research paper (arXiv:2603.06728, March 2026) introduces **Orion** — the first open system combining direct ANE execution, a compiler pipeline, and stable multi-step training with checkpoint resume, bypassing CoreML entirely via Apple's private `_ANEClient` and `_ANECompiler` APIs. The paper catalogs 20 restrictions on MIL IR programs (14 previously undocumented), introduces an 8.5x recompilation speedup trick (4,200ms → 494ms per step), and achieves 170+ tokens/s for GPT-2 124M inference on M4 Max. Also presents LoRA adapter-as-input enabling hot-swap of adapters without recompilation. **This is major — ANE training is now demonstrated and documented.**
- [arXiv](https://arxiv.org/abs/2603.06728)
- [arXiv PDF](https://arxiv.org/pdf/2603.06728v1)

## ⚡ ANE Innovation Watch

### Orion: ANE Training + Inference — arXiv:2603.06728
**The headline ANE development this period.** Orion (March 2026) is the first documented open system achieving end-to-end ANE training via Apple's private APIs. Key metrics: 170+ tok/s GPT-2 124M on M4 Max; stable 110M transformer training on TinyStories (1,000 steps in 22 min, zero NaN); 3.8x training speedup via weight patching technique (494ms/step vs naive 4,200ms). Catalogued 20 ANE restrictions (14 previously unknown). LoRA adapter hot-swap without recompilation demonstrated.

## 📝 BEP Research Insight (from "The Other Memory Wall, Part 2")

Ben Pouladian's latest BEP Research post (March 2026) synthesizes four data points into a cohesive thesis: **Apple's three-layer AI inference architecture is a strategic moat.**

1. **M5 Pro/Max chiplet architecture** — Fusion Architecture connects two dies; Neural Accelerators in each GPU core; same CPU die across Pro/Max/Ultra enabling modular scaling from laptop to PCC server
2. **Qwen3.5-397B at 38 tok/s on M3 Ultra** via MLX (Ivan Fioravanti) — 397B MoE model at conversational speed on two-generation-old hardware
3. **Qwen 3.5 2B at 6-bit on iPhone 17 Pro** via MLX (Adrien Grondin) — viral post (1.8M views), amplified by Alibaba's own account
4. **PCC upgraded to M5, skipping M3/M4 entirely** — signals strategic priority; Agent Worker in iOS 26.4 enables seamless on-device/cloud orchestration

**Core thesis:** Apple has one unified memory architecture spanning A19 Pro (iPhone 17 Pro) → M5 (MacBook Pro/Mac Studio/Mac Pro) → M5 (PCC servers). Same LPDDR interface. Same MLX. Same zero-copy memory access patterns. No other company has this — Google has TPUs + Tensor chips (different architectures), Microsoft has NVIDIA + Qualcomm (completely different stacks), Amazon has Trainium (no device platform).

**Privacy as infrastructure:** Apple sells hardware, not data. Layer 1 data never leaves device. Layer 2 (PCC) processed in hardware-isolated secure enclaves, immediately discarded. Layer 3 (Gemini) is explicit opt-in. "Your data shouldn't train someone else's model" is an architectural commitment, not just marketing.

**Key quote:** "NVIDIA owns one side of the memory wall. Apple is building a moat on the other. And that moat is three layers deep."
- [BEP Research Substack](https://bepresearch.substack.com/p/the-other-memory-wall-part-2)
